Batch thumbnail downloads via a CDN bundle endpoint

Background (Why?)

The catalog service builds product thumbnails by downloading each image from our CDN one at a time. That's fine for a single product page, but it falls apart for bulk jobs: rebuilding a whole category, or importing a new supplier's catalog, means asking the CDN for an image every single time, one HTTP request after another.

The CDN rate-limits that pattern. Past a certain number of requests in a short window it starts answering with 429, and the rebuild aborts partway through the category, leaving thumbnails half-generated with no easy way to know where it stopped.

Pending

Description (How?)

The core idea The CDN stores images in per-category folders and can hand back the whole folder as one .zip via a ?bundle endpoint. So instead of asking for each image one at a time, ask for the category once, unpack it locally, and only fall back to individual requests for whatever the archive didn't have.
Why a threshold, not always-bundle Bundle mode only kicks in once a job asks for more than 2 images. A 1–2 image request — the per-page case — isn't worth building and unpacking a .zip for, so it keeps using plain per-image requests exactly as before. Nothing about the single-page path changes.
